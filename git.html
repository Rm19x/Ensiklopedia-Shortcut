Daftar Lengkap Shortcut & Perintah Cepat Git

Perintah / Shortcut CLI Fungsi / Deskripsi Nyata
git init Initialize Repository: Membuat repositori Git baru lokal di dalam folder proyek aktif.
git clone <url> Clone Repository: Mengunduh atau menggandakan repositori jarak jauh (seperti dari GitHub) ke komputer lokal.
git config --global user.name "Nama" Set Username: Mengonfigurasi nama pengguna secara global untuk riwayat commit.
git status Check Status: Memeriksa status berkas (apakah ada file yang dimodifikasi, dihapus, atau belum dilacak).
git add . Stage All Changes: Memasukkan seluruh berkas yang baru atau yang mengalami perubahan ke dalam staging area (siap di-commit).
git add <nama-file> Stage Specific File: Hanya memasukkan berkas spesifik tertentu ke dalam staging area.
git commit -m "Pesan" Commit with Message: Menyimpan perubahan yang ada di staging area ke dalam riwayat Git lokal lengkap dengan pesan catatan.
git commit --amend -m "Pesan Baru" Amend Last Commit: Mengubah pesan commit terakhir atau menggabungkan perubahan baru ke dalam commit paling buncit tanpa membuat commit baru.
git branch List Branches: Menampilkan daftar seluruh branch (cabang) lokal yang ada di dalam proyek.
git branch <nama-cabang> Create Branch: Membuat cabang alur kerja baru tanpa langsung berpindah ke cabang tersebut.
git checkout <nama-cabang> Switch Branch: Berpindah fokus kerja ke cabang/branch lain yang dituju.
git checkout -b <nama-cabang> Create & Switch Branch: Jalan pintas tercepat untuk membuat cabang baru sekaligus langsung berpindah ke dalamnya.
git merge <nama-cabang> Merge Branch: Menggabungkan riwayat perubahan dari cabang tujuan ke dalam cabang aktif tempat kamu berada saat ini.
git branch -d <nama-cabang> Delete Branch: Menghapus cabang lokal yang sudah selesai digunakan atau sudah di-merge secara aman.
git remote add origin <url> Link Remote: Menghubungkan repositori lokal milikmu ke repositori server jarak jauh (seperti repositori di GitHub).
git push -u origin <nama-cabang> First Push: Mengunggah hasil kerja lokal ke server jarak jauh untuk pertama kalinya sekaligus mengatur upstream pelacakan.
git push Regular Push: Mengunggah commit baru lokal ke server remote (jika upstream sudah diatur sebelumnya).
git pull Pull Changes: Mengambil sekaligus menggabungkan (fetch + merge) commit terbaru dari server jarak jauh ke proyek lokalkan agar sinkron.
git fetch Fetch History: Hanya mengambil info riwayat perubahan terbaru dari server jarak jauh tanpa langsung menimpanya ke berkas lokal.
git log --oneline Compact History Log: Menampilkan riwayat commit proyek secara ringkas (satu baris per commit) agar mudah dibaca cepat.
git reset --soft HEAD~1 Undo Last Commit Safely: Membatalkan satu commit terakhir, tetapi tetap menjaga berkas hasil modifikasimu aman di staging area.
git reset --hard HEAD~1 Hard Reset Last Commit: Membatalkan commit terakhir dan MENGHAPUS total semua perubahan berkas dokumen yang ada (Hati-hati, data hilang permanen!).
git stash Stash Changes: Menyimpan sementara perubahan pengerjaan berkas yang belum siap di-commit ke dalam kantong penyimpanan darurat, membuat direktori kembali bersih.
git stash pop Restore Stash: Mengambil kembali perubahan pengerjaan yang sebelumnya disimpan di dalam kantong darurat stash ke berkas aktif saat ini.
